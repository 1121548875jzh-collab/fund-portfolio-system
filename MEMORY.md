## 经验
- **Tushare净值更新时间**: 隔日早上8:20后
- **执行顺序**: 净值优先→所有依赖任务在净值更新后执行
- **T+1确认**: 今天买入，明天确认
- **份额计算**: 份额 = 金额 / 交易日净值（⚠️必须用交易日净值，不能用Tushare最新净值）
- **净值日期校验**: Tushare返回的净值日期必须匹配交易日，否则跳过等待
- **QDII净值延迟**: T+1延迟，报告日期=普通基金最新净值日期，QDII当日盈亏用(T日-T-1日)净值

## 快照规则 ⚠️核心
**执行时间**: 8:20（净值更新后）

**快照流程**:
1. 从Tushare获取最新净值 → 更新fund_nav_history表
2. 确认PENDING交易 → 更新fund_holdings表
3. 从fund_holdings读取数据 → 生成快照到daily_fund_snapshot表
4. 发送邮件报告

**核心规则**:
- 每日快照独立保存，日期唯一
- 快照数据来源：fund_holdings表（手动修正后以holdings为准）
- 当日盈亏计算：
  - 普通基金：份额 × (今日净值 - 昨日净值)
  - QDII基金：份额 × (今日净值 - 昨日净值)，日期延迟一天
- 报告日期 = 普通基金最新净值日期

**数据表关系**:
```
fund_holdings (当前持仓，实时更新)
     ↓ 每日快照
daily_fund_snapshot (历史快照，按日期独立)
```
