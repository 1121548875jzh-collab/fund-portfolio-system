# memory-preserve - 压缩前保存记忆

在执行 `/compact` 前，主动保存当前对话的关键信息到记忆文件，确保压缩后不丢失重要内容。

**命令**: `/save-memory`

---

## 功能

| 子命令 | 功能 |
|:---|:---|
| `/save-memory` | 保存当前对话要点 + 提示压缩 |
| `/save-memory force` | 强制保存（忽略去重检查） |

---

## 执行流程

1. **分析对话** - 提取关键决策、待办、学习要点
2. **写入日常记忆** - 更新 `memory/YYYY-MM-DD.md`
3. **更新长期记忆** - 更新 `MEMORY.md`（如有重要决策）
4. **返回报告** - 显示保存内容摘要
5. **提示压缩** - 建议发送 `/compact`

---

## 记忆写入规则

### memory/YYYY-MM-DD.md（日常笔记）

```markdown
## YYYY-MM-DD HH:MM - 对话摘要

### 关键决策
- 决策1
- 决策2

### 待办事项
- [ ] 待办1
- [ ] 待办2

### 学习要点
- 要点1
- 要点2
```

### MEMORY.md（长期记忆）

仅更新重要内容：
- 系统状态更新
- 能力缺口变更
- 经验教训
- 项目进度

---

## Agent 执行步骤

当收到 `/save-memory`：

```markdown
1. 回顾当前对话，提取关键信息
2. 使用 edit 工具更新 memory/YYYY-MM-DD.md
3. 如有重要决策，更新 MEMORY.md
4. 返回保存摘要
5. 提示：💡 现可发送 /compact 压缩上下文
```

---

## 与 memoryFlush 的关系

| 功能 | 触发时机 | 作用 |
|:---|:---|:---|
| `/save-memory` | 手动触发 | 主动保存，用户可控 |
| `memoryFlush` | 自动压缩前 | 自动保存，静默执行 |

**推荐流程**：
```
/save-memory → 主动保存重要信息
/compact      → 系统压缩上下文
              → memoryFlush 自动补充保存
```

---

## 注意事项

- 保存前检查是否已存在类似内容（避免重复）
- 使用 edit 工具追加内容，不覆盖已有记忆
- NO_REPLY 模式不适用于此命令（用户需看到保存结果）
